CRITICAL FINDINGS
================================================================================

1. SERPER API LIMITATION
   ❌ Does NOT provide publish dates in organic search results
   ✓ DOES provide dates in news results (if switching API)
   → Need to either parse HTML or switch API endpoint

2. DATABASE READY
   ✓ Already has first_seen_at (good proxy)
   ✓ Can add published_at field later without breaking existing data
   ✓ No urgent changes needed

3. FRONTEND READY
   ✓ ArticleCard component already structured for additional metadata
   ✓ Can add date display with minimal changes
   ✓ CSS flexbox layout accommodates new element

4. NO BLOCKING ISSUES
   ✓ Can implement Phase 1 (frontend only) immediately
   ✓ Phase 2 (full implementation) can wait for better date source

================================================================================

QUICK DECISION MATRIX
================================================================================

Want dates ASAP?
  → Go with Phase 1 (2-3 hours)
  → Users see "Jan 15" but it means "we found it on Jan 15"
  → Can upgrade later to actual publish dates

Want accurate dates?
  → Need Phase 2 (4-6 hours)
  → Must parse HTML from articles OR switch to News API
  → Better long-term solution

Want maximum accuracy?
  → Combine Phase 1 + Phase 2
  → Show dates from Serper if available, else fetch from article HTML
  → Most work but best user experience
